如何使一个类不可变

  1. 将类声明为 final，使其无法被继承

  2. 所有域都使用 private 进行修饰，不允许被访问

  3. 不提供变量的 setter 方法

  4. 所有可变域都使用 final 修饰，使它的值只能被分配一次

  5. 通过构造函数进行深克隆初始化所有域

  6. 对 getter 方法获取的对象执行克隆以返回副本，而不是实际对象引用

为什么 Stirng 类是不可变的

String 能被设计成不可变类型的一个重要前是因为它是编程语言里面使用频率最高的一种类型。不可变类型带来的好处，体现在四个方面，分别是：缓存，安全，同步和性能

为什么成员变量会赋默认值，局部变量需要显示赋值

成员变量的赋值是类加载的过程中赋予了默认值；另外成员变量不确定什么时候会被取值使用，也有可能在各个方法中赋不用的值，所以显式的赋值意义不大，默认即可

首先 java 规定，局部变量需要显式的赋值

理由：局部变量运行时被分配在栈中，量大、生命周期短，如果虚拟机给每个局部变量都初始化，对于性能而言也是一笔不小的开销。并且局部变量不进行初始化，使用默认值使用是不安全的。处于性能以及安全两方面考虑，解决方案是虚拟机不进行初始化操作，交由编写者进行手动赋值

重写和重载的区别

重写的特征

  1. 参数列表必须与被重写的方法把持一致

  2. 返回 的类型必须与被重写的方法保持一致

  3. 访问权限修饰符一定要大于被重写方法

  4. 重写方法一定不能抛出新的检查异常或者比被重写方法申请更加广泛的检查型异常

父类的一个被重写方法申请了一个检查异常 IOException，在重写这种方法时就不能抛出 Execption，仅仅能抛出 IOExecption 的子类异常。能够抛出非检查型异常

重载的特征

  1. 必须具备不同的参数列表

  2. 能够有不同的返回类型。仅仅参数列表不同就行

  3. 能够有不同的访问修饰符

  4. 能够抛出不同的异常

hashCode 与 equals 区别

Java 对于 eqauls 方法和 hashCode 方法是这样规定的：

  1. 如果两个对象相同（equals 方法返回 true），那么它们的 hashCode 值一定要相同；

  2. 如果两个对象的 hashCode 相同，它们并不一定相同。

实现高质量的 equals 方法的诀窍包括：

  1. 使用==操作符检查”参数是否为这个对象的引用”

  2. 使用 instanceof 操作符检查”参数是否为正确的类型”

  3. 对于类中的关键属性，检查参数传入对象的属性是否与之相匹配

  4. 编写完 equals 方法后，问自己它是否满足对称性、传递性、一致性

  5. 重写 equals 时总是要重写 hashCode

  6. 不要将 equals 方法参数中的 Object 对象替换为其他的类型，在重写时不要忘掉@Override 注解
